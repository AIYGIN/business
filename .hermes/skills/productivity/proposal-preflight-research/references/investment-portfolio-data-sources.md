# 投資ポートフォリオ/ETF・投信データ提案の調査メモ

投資ポートフォリオ画面、ETF/投信の中身可視化、セクター・国地域・上位構成銘柄分析の提案前調査で再利用する要点。

## 典型的な結論

v1は「対象商品を限定した商品マスタ + 月次/半手動更新」から始めるのが現実的。いきなり外部API全面依存やPDFスクレイピング自動化に寄せると、データライセンス、再配信条件、国内投信カバレッジ、PDF様式差分、保守コストが重くなる。

## データ取得候補

| 候補 | 使いどころ | 注意点 |
|---|---|---|
| FMP ETF & Fund Holdings API | ETF/投信の保有銘柄・国別・セクター等のAPI候補 | 有料プラン、表示・再配信・キャッシュ条件の確認が必要 |
| Alpha Vantage | 低頻度PoC/API検証 | 無料枠は小さい。商用/リアルタイム/権利条件は要確認 |
| API Ninjas ETF API | ETF holdings、expense ratio、AUM等 | holdings等はpremium onlyの項目。キャッシュ/保存条件確認が必要 |
| Finnhub ETF API | ETF holdings/profile/exposure候補 | ETF/indices向けは高額になり得る。国内投信カバレッジは要確認 |
| Morningstar / LSEG | 本格商用・広範囲ファンドカバレッジ | 法人向け見積/契約前提。小規模v1には重い可能性 |
| 運用会社の月次レポート | 国内投信の上位10銘柄、国地域、業種比率 | PDF中心。自動抽出は様式差分・URL変更・精度検証が重い |
| EDINET | 法定開示資料の取得 | v1画面用の最新構成比率抽出には解析/保守コストが高い |

## 既存サービス/代替手段

- Morningstar Portfolio X-Ray: 複数ETF/投信を含むポートフォリオを資産クラス、セクター、地域、費用などで分析する既存機能。
- Portfolio Visualizer: ポートフォリオ分析、バックテスト、AI説明などを提供する既存サービス。

既存サービスがある場合も、内製の価値は「特定UX」「SBI等への外部確認導線」「日本語の注意喚起」「限定した投資方針との整合チェック」などに置くとよい。

## v1スコープ推奨

- 保有商品と投資比率は手入力。
- 評価額、損益、SBIログイン連携、リアルタイム株価、売買推奨、AI銘柄推奨は除外。
- 対象商品は主要ETF/投信に限定する。
- 商品構成データは source_url、as_of_date、更新者/更新方法を持つCSV/JSON/DBマスタに正規化する。
- 画面表示ごとに外部APIを叩かず、日次/月次バッチまたは手動更新結果をキャッシュ/保存する。

## リスク観点

- データ表示・再配信・キャッシュ・加工表示のライセンス条件。
- 投資助言/売買推奨と誤認される表現。
- データ鮮度・転記ミス・計算誤差。
- セクター分類体系の違い。
- 国内投信のマザーファンド/ファンド・オブ・ファンズ構造。
- API障害時のフォールバックと古いデータ警告。

## 確認ゲートで聞くべきこと

1. 用途は個人/社内/一般公開/商用SaaSのどれか。
2. v1対象商品はどこまで限定するか。
3. データ更新頻度は月次で許容できるか。

## 調査時の実務メモ

サブエージェント調査がタイムアウトした場合でも、親エージェントは停止せず、直接 web_search/web_extract で主要論点を補完し、どの部分が補完調査でどの部分が未確認かを分けて確認ゲートに出す。タイムアウトを事実として記録し、サブエージェントの未完了結果を事実扱いしない。
